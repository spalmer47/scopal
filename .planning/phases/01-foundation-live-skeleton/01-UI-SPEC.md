# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation + Live Skeleton phase. Generated by gsd-ui-researcher (auto mode), to be verified by gsd-ui-checker.

**Scope reminder:** Phase 1 ships a placeholder homepage, `BaseLayout` (Header + Footer with all legal disclaimers), `MarketingLayout`, security headers, CI rails, `constants.ts`, and `content.config.ts`. No real homepage copy, no practice area content, no contact form, no mobile menu (D12 — zero JS by default; mobile menu deferred to Phase 2 when real nav content exists).

This contract locks visual tokens that all later phases inherit. Phases 2–4 extend the system; they do not redefine it.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Tailwind v4 CSS-first config; no shadcn) |
| Preset | not applicable — Astro project per D2; shadcn is React-only and project ships zero JS islands (D12) |
| Component library | none (Astro components authored in-repo) |
| Icon library | inline SVG only in Phase 1 (no icon font, no icon library); deferred to Phase 2+ if needed (lucide-static recommended when icons are introduced) |
| Font | self-hosted woff2 in `public/fonts/` per D10 — max 2 families × 2 weights, `font-display: swap`, above-fold weight preloaded |

**Rationale:** shadcn was considered and rejected because (a) the stack is Astro with zero React/Vue islands per D12, (b) shadcn's primitives are React/Radix-based, and (c) the placeholder homepage and BaseLayout chrome do not need a component library. Tailwind v4 `@theme {}` tokens in `src/styles/global.css` are the design system primitive for this project.

## Spacing Scale

Tailwind v4 defaults are an 8-point/4-point hybrid scale (`0.25rem` step). We adopt the standard scale and declare the named tokens this phase uses. All values are multiples of 4.

| Token | Value | Tailwind class | Usage in Phase 1 |
|-------|-------|----------------|------------------|
| xs | 4px | `p-1` / `gap-1` | Inline icon-to-text gap (when icons land in Phase 2) |
| sm | 8px | `p-2` / `gap-2` | Footer disclaimer line spacing, compact inline groups |
| md | 16px | `p-4` / `gap-4` | Default body paragraph spacing, header inner padding (mobile) |
| lg | 24px | `p-6` / `gap-6` | Header inner padding (desktop), footer column gap |
| xl | 32px | `p-8` / `gap-8` | Section vertical padding (mobile) |
| 2xl | 48px | `p-12` / `gap-12` | Section vertical padding (desktop), footer top padding |
| 3xl | 64px | `p-16` / `gap-16` | Hero/landing vertical breathing room on placeholder homepage |

**Container:** Page content max-width `1200px` (Tailwind `max-w-screen-xl` ≈ 1280px is acceptable; choose `max-w-6xl` = 1152px or set `--container-page: 1200px` in `@theme`). Horizontal page padding: `16px` mobile, `24px` tablet, `32px` desktop.

**Touch target minimum:** 44×44px for any interactive element (WCAG 2.1 AA target). The single placeholder CTA must meet this — vertical padding ≥ `12px`, horizontal padding ≥ `20px` at the declared body font size yields ~44px.

**Exceptions:** none in Phase 1.

## Typography

Two families, two weights total (D10 cap). Type scale picks 4 sizes — no more.

| Property | Value |
|----------|-------|
| Body family | `Inter` (self-hosted woff2), fallback `system-ui, -apple-system, "Segoe UI", sans-serif` |
| Display family | `Fraunces` (self-hosted woff2), fallback `Georgia, "Times New Roman", serif` |
| Body weight | 400 (regular) |
| Display/strong weight | 600 (semibold) — used for both display headings and inline `<strong>` so we stay within the 2-weight cap |
| Preloaded above-fold weight | Inter 400 (the placeholder homepage is body-heavy) |
| `font-display` | `swap` |

**Font choice rationale:** Inter is a neutral, highly legible business sans that survives at small sizes (footer disclaimers). Fraunces is a modern variable serif with optical sizing — it provides the "retro-modern, playful, approachable" personality called out in FIRM_BRIEF (newfangled.legal reference) without sacrificing the trust-signaling weight a law firm needs. Both are open-licensed and self-hostable. If Scott prefers a tighter, more conventional pairing later, `Fraunces` can be swapped for `Playfair Display` (mentioned in RESEARCH.md Pattern 2) without touching layout.

**Type scale (mobile-first; desktop nudges in parentheses where it changes):**

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body | 16px (1rem) | 400 | 1.5 (24px) | `text-base leading-relaxed` |
| Label / footer fine print | 14px (0.875rem) | 400 | 1.5 (21px) | `text-sm leading-relaxed` |
| Heading (h2/h3) | 24px mobile / 28px desktop | 600 | 1.25 | `text-2xl md:text-[28px] font-semibold` |
| Display (h1, hero) | 36px mobile / 48px desktop | 600 | 1.15 | `text-4xl md:text-5xl font-semibold` |

**Constraints:**
- Body line-height 1.5 is the legal-readability floor; do not tighten.
- Display line-height ≤ 1.2 to keep multi-line headings cohesive.
- Maximum line length for prose: ~70ch (Tailwind `max-w-prose`).
- Letter-spacing: default for all roles; do NOT add tracking adjustments in Phase 1.
- The 14px label size is reserved for footer disclaimers, copyright, and future form helper text. Body copy never goes below 16px.

## Color

Approach: **clean modern base, orange + blue used as small-dose accents** per FIRM_BRIEF. Not a color wash. The dominant 60% is white/near-white surface; the secondary 30% is a deep ink for text and dark footer; the 10% accent splits between brand-blue (primary action) and brand-orange (highlight/personality). All declared in OKLCH per RESEARCH.md Pattern 2.

| Role | Token | Value (OKLCH) | Approx hex | Usage |
|------|-------|---------------|------------|-------|
| Dominant (60%) | `--color-surface` | `oklch(1 0 0)` | `#FFFFFF` | Page background |
| Dominant alt | `--color-surface-muted` | `oklch(0.985 0.005 240)` | `#F8FAFC` | Optional alternate section background (used sparingly) |
| Secondary (30%) | `--color-ink` | `oklch(0.22 0.04 250)` | `#1A2230` | Body text, footer background (inverted), heading color |
| Secondary alt | `--color-ink-muted` | `oklch(0.45 0.02 250)` | `#5B6675` | Footer fine print, labels, secondary text |
| Border / divider | `--color-border` | `oklch(0.92 0.01 240)` | `#E6E8EC` | Hairlines, dividers, focus-ring offset |
| Accent — primary | `--color-brand-blue-500` | `oklch(0.50 0.16 240)` | ~`#1D5FBF` | Primary CTA background, active link, focus ring |
| Accent — primary hover | `--color-brand-blue-600` | `oklch(0.42 0.17 240)` | ~`#164D9C` | CTA hover/active state |
| Accent — secondary | `--color-brand-orange-500` | `oklch(0.70 0.18 50)` | ~`#E97B1F` | Brand wordmark accent dot/underline; reserved highlight only |
| Destructive | `--color-destructive-600` | `oklch(0.52 0.22 27)` | ~`#C8392E` | Reserved for Phase 4 form errors; not used in Phase 1 |

**Accent reserved for (explicit list — Phase 1):**
1. Primary CTA button background ("Schedule a Consultation") — `brand-blue-500`
2. Inline text links inside the footer disclaimer block (privacy/terms once they exist; Phase 1 footer has no internal links yet, so this is dormant) — `brand-blue-500`
3. Keyboard focus ring — `brand-blue-500` at 2px outer with `border` color offset
4. Brand wordmark accent — single underline or accent dot beside "Scopal Firm" in the header — `brand-orange-500`

Accent is NEVER used for: body copy, large background fills, decorative gradients, or "all interactive elements." If a future element needs interactivity in this phase, default to `--color-ink` for text + underline on hover.

**Contrast verification (WCAG 2.1 AA):**
- Body `--color-ink` on `--color-surface`: contrast ≥ 12:1 (passes AA + AAA for body)
- Footer `--color-ink-muted` on `--color-surface`: must be ≥ 4.5:1 — verified on the chosen OKLCH lightness; if axe-core reports a fail, darken `--color-ink-muted` by 0.05 lightness
- White text on `--color-brand-blue-500`: ≥ 4.5:1 required for the CTA — verified at L=0.50; if it fails, drop to `--color-brand-blue-600`
- Footer (inverted): `--color-surface` text on `--color-ink` background — ≥ 12:1
- Focus ring on white background: `--color-brand-blue-500` ring ≥ 3:1 against `--color-surface` ✓

**Pitfall guard:** Never use `--color-brand-orange-500` on white for body copy or links — the contrast at L=0.70 fails AA for normal text. Orange is decorative-only in v1.

### Banned vocabulary (D14 — CI-enforced)

The following terms must NEVER appear in any page, layout, component, or content collection in Phase 1: `specialist`, `specializing in`, `expert`, `experts`, `the best`, `#1`, `leading`, `top-rated`, `super lawyer`. Permitted alternatives: `focuses on`, `represents clients in`, `experienced in`. Additionally, the strings `Contact Us` and `Free Consultation` are forbidden as CTA labels (HOME-05; D16). The CTA in Phase 1 says `Schedule a Consultation` — never `Book a Fit Call` in Phase 1 (reserve that variant for Phase 3 pricing context where it lands more naturally).

## Layout & Interaction Contract

### BaseLayout structure

```
<html lang="en">
  <head> [SEO + JSON-LD LegalService on / only] </head>
  <body class="min-h-screen flex flex-col bg-surface text-ink">
    <a href="#main" class="sr-only focus:not-sr-only ...">Skip to content</a>
    <Header />
    <main id="main" class="flex-1"> <slot /> </main>
    <Footer />
  </body>
</html>
```

**Required for accessibility (axe-core CI gate):**
- `lang="en"` on `<html>`
- Skip-to-content link, visually hidden until focused, lands on `<main id="main">`
- Single `<h1>` per page
- Landmark elements: `<header>`, `<main>`, `<footer>` — no `<div role="banner">` substitutes
- Color contrast ≥ 4.5:1 for all text (verified above)
- Focus visible on every interactive element — 2px `--color-brand-blue-500` outline with 2px offset

### Header

- Sticky? **No** in Phase 1. (Sticky CTA pattern lands in Phase 2 with HOME-05 persistent CTA requirement.)
- Height: `64px` mobile, `80px` desktop
- Layout: brand wordmark left-aligned; CTA right-aligned on desktop (≥768px); CTA hidden on mobile in Phase 1 (mobile menu deferred per D12)
- Background: `--color-surface` with bottom border `1px solid --color-border`
- Wordmark: Display font (Fraunces) at 20px mobile / 24px desktop, weight 600. Optional `--color-brand-orange-500` accent dot or 2px underline on the final letter — restrained, not garish

### Footer

- Background: `--color-ink` (dark)
- Text: `--color-surface` for primary, `oklch(0.78 0.01 240)` (a lightened ink variant) for fine-print disclaimers — verify ≥ 4.5:1 against the dark background
- Layout: single column on mobile, two-column on desktop (left: NAP + responsible attorney + bar status; right: disclaimers stack)
- Vertical padding: `48px` top, `32px` bottom
- Typography: `text-sm` (14px) for all footer text; copyright line at the very bottom centered or left-aligned with a `1px solid` divider above
- All copy strings pulled from `src/lib/constants.ts` — never hard-coded in `Footer.astro`

### Placeholder Homepage layout

- Single centered column, `max-w-prose` (~65ch)
- Vertical padding: `64px` top, `48px` bottom on mobile; `96px` / `64px` on desktop
- H1 → tagline (display 24px regular weight italic-optional) → body paragraph → CTA button — in that order, with `24px` between blocks
- CTA button: `--color-brand-blue-500` background, white text, `12px 20px` padding, `8px` border radius, body font 16px weight 600; hover → `--color-brand-blue-600`; focus → 2px white inner ring + 2px blue outer ring

### Interaction states (locked for the project)

| State | Treatment |
|-------|-----------|
| Default link | `--color-ink` text, underline on (text-decoration-thickness 1px, underline-offset 2px) |
| Hover link | `--color-brand-blue-500` text, underline persists |
| Focus (keyboard) | 2px solid `--color-brand-blue-500` outline, 2px offset, no outline removal anywhere |
| Disabled (none in P1) | reserved: 50% opacity + `cursor-not-allowed` |
| Active CTA | `--color-brand-blue-600` background |

### Motion

Phase 1 ships **no animations, no transitions beyond browser defaults**. Reasoning: zero-JS constraint (D12), Lighthouse CLS budget < 0.05, and the placeholder is too thin to warrant motion design. Hover color changes use Tailwind's default ~150ms transition only. Honor `prefers-reduced-motion: reduce` from day one (set in `global.css` base layer to disable any future transitions globally for users who request it).

| Breakpoint | Width | Used for |
|------------|-------|----------|

| `lg:` | ≥ 1024px | Desktop: type scale upper bounds (h1 48px, h2 28px), section padding bumps to 32px/64px |

`xl:` and `2xl:` not used in Phase 1.

## Open UI Decisions Deferred to Later Phases

These are intentionally NOT specified in Phase 1 and must be decided when their phase begins:

- Mobile menu pattern (Phase 2 — when nav has real destinations)
- Sticky CTA / scroll-aware header (Phase 2 — HOME-05)
- Hero imagery, photography style, illustration approach (Phase 2 — HOME-01..04)
- Trust strip layout (Phase 2 — HOME-04)
- Card/list patterns for practice areas, attorney bio, pricing tiers (Phase 3)
- Form field, validation, and inline-error visual language (Phase 4 — FORM-01..06)
- 404 / legal pages visual treatment (Phase 4 — LEGAL-03)
- OG default image (Phase 4 — SEO-03)

This deferral is deliberate — Phase 1's job is to lock the *foundation* tokens (color, type, spacing, layout chrome, copy contract) so later phases inherit a stable system instead of redesigning each time.
