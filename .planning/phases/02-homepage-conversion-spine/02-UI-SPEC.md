# Phase 2 — UI Design Contract

> Visual and interaction contract for the Homepage + Conversion Spine phase. Generated by gsd-ui-researcher, to be verified by gsd-ui-checker.

**Scope reminder:** Phase 2 ships the full StoryBrand homepage (`src/pages/index.astro`), a placeholder contact page (`src/pages/contact.astro`), an updated `Header.astro` with real nav + mobile hamburger, new homepage section components (Hero, ProactiveInsight, TrustStrip, Plan, Stakes, SuccessVision, CTASection), and a new `WebSiteSchema.astro` JSON-LD component.

**Inheritance:** All color tokens, spacing scale, typography families/weights/sizes, container width, and interaction states are LOCKED from Phase 1 (`01-UI-SPEC.md`). This document extends that system with Phase 2-specific section layout contracts. It does NOT redefine base tokens.

## Spacing Scale

Inherited from Phase 1 (all multiples of 4). Phase 2 usage added in the rightmost column.

| Token | Value | Tailwind class | Phase 2 Usage |
|-------|-------|----------------|---------------|
| xs | 4px | `p-1` / `gap-1` | Icon-to-text gap; step number to step label |
| sm | 8px | `p-2` / `gap-2` | Trust strip logo internal padding; nav link horizontal gap |
| md | 16px | `p-4` / `gap-4` | Body paragraph spacing; hero sub-headline margin-top; mobile section horizontal padding |
| lg | 24px | `p-6` / `gap-6` | Section heading to body gap; step card internal padding; trust strip group label to logo row gap |
| xl | 32px | `p-8` / `gap-8` | Between-section vertical gap on mobile; CTA section internal padding |
| 2xl | 48px | `p-12` / `gap-12` | Section vertical padding desktop; hero section top padding mobile |
| 3xl | 64px | `p-16` / `gap-16` | Hero section top padding desktop; page-end CTA section vertical padding |

**Exceptions:**
- Mobile hamburger menu touch target: minimum 44×44px per WCAG 2.1 AA (button padding to reach 44px height; standard `p-3` at 16px base = 48px — use `p-3` on the hamburger button)
- Trust strip logos: height capped at 32px (SVG) / 40px (PNG) to maintain visual rhythm regardless of source logo aspect ratio
- Step number circles: 48×48px fixed (12px × 4 = 48) — this is a decorative container, not a touch target

## Typography

Inherited from Phase 1 (2 families × 2 weights). Phase 2 maps each role to specific homepage elements.

| Role | Size | Weight | Line Height | Tailwind | Phase 2 Applied To |
|------|------|--------|-------------|----------|--------------------|
| Body | 16px (1rem) | 400 (Inter) | 1.5 | `text-base leading-relaxed` | Hero sub-headline, section body paragraphs, step descriptions, contact placeholder body, nav link text |
| Label | 14px (0.875rem) | 400 (Inter) | 1.5 | `text-sm leading-relaxed` | Trust strip group labels ("Prior in-house experience" / "Professional associations"); nav items on mobile; step number label ("Step 1" etc.) |
| Heading | 24px mobile / 28px desktop | 600 (Fraunces) | 1.25 | `text-2xl md:text-[28px] font-semibold` | Section headings (ProactiveInsight, Plan, Stakes, SuccessVision, CTA sections); 3-step plan step names |
| Display | 36px mobile / 48px desktop | 600 (Fraunces) | 1.15 | `text-4xl md:text-5xl font-semibold` | Hero H1 only: "You need a GC, but not the salary that comes with it." |

**Phase 2-specific typography rules:**
- Hero H1 (`text-4xl md:text-5xl`): Fraunces 600, color `--color-ink` (`#1A2230`), line-height 1.15. Maximum 2 lines on desktop. Do not apply `--color-brand-orange-500` to any word in the H1 — accent is reserved for decorative elements only.
- Hero sub-headline: Inter 400, 16px (body role), color `--color-ink-muted` (`#5B6675`), max-width `max-w-prose` (~65ch), margin-top `lg` (24px).
- Section headings: Fraunces 600, heading role, color `--color-ink`. No section heading uses display size — display is reserved for the H1.
- Step names in the 3-step Plan section use the heading role (24px/28px). The "Step N:" prefix label uses the label role (14px, `--color-ink-muted`).
- CTA button text: Inter 600, 16px — same as body weight but at semibold (weight 600 is within the 2-weight cap because it is the same weight used for display/headings).
- Nav link text: Inter 400, 16px (body role). Active/current page: weight stays 400, color shifts to `--color-brand-blue-500`.
- Letter-spacing: default for all roles. No tracking adjustments introduced in Phase 2.
- Maximum prose line length: `max-w-prose` (~65ch) for all body copy blocks. Section headings: `max-w-2xl` on desktop.

**Accent reserved for (complete list — Phase 2):**
1. Primary CTA button background ("Book a Fit Call") — `brand-blue-500`

If scroll-aware shadow adds complexity, ship with a static `1px solid --color-border` bottom border (same as Phase 1) — this is acceptable.

**Height:** 64px mobile / 80px desktop (inherited from Phase 1).

**Layout (desktop, ≥768px):**
```
[Logo/wordmark]   [Nav: Practice Areas · Scott Palmer · Pricing]   [Book a Fit Call ▶]
left              center-right                                       far right
```
- Firm wordmark: left-aligned. Fraunces 600, 20px mobile / 24px desktop. Optional `--color-brand-orange-500` accent dot or 2px underline on last letter of "Scopal Firm" (inherited Phase 1 decision).
- Nav links: `Practice Areas`, `Scott Palmer`, `Pricing` — center/right cluster. Inter 400, 16px. Color `--color-ink`. Hover → `--color-brand-blue-500`. Active → `--color-brand-blue-500`.
- CTA button (desktop): "Book a Fit Call" — primary button style (see Button Variants below). Links to `/contact`.
- Header bottom border: `1px solid --color-border` (default, non-scrolled state).

**Layout (mobile, <768px):**
```
[Logo/wordmark]                           [☰ hamburger button]
left                                      right
```
- Nav links hidden by default. Hamburger button: 44×44px touch target, `--color-ink` icon (3 horizontal bars, 20px wide × 2px thick × 6px gap). Implemented with vanilla `<script>` per D-16. No "Book a Fit Call" CTA button in the header on mobile — CTA appears inside the open mobile nav drawer and in page sections.
- **Mobile nav open state:** Full-width panel slides or fades below the header (not a full-screen overlay). Background: `--color-surface`. Border-bottom: `1px solid --color-border`. Contains: nav links stacked vertically (`gap-6` / 24px between links), then "Book a Fit Call" button (full-width, primary style). Nav links: Inter 400, 16px, `--color-ink`, `padding: 12px 0` for tap target height.
- **Hamburger open/close state:** Toggle `aria-expanded` on the button. Toggle `hidden`/`block` (or `max-h-0`/`max-h-screen` with CSS transition) on the nav panel. No JS framework — one `<script>` block in `Header.astro`.
- **Accessibility:** `<nav aria-label="Main navigation">` wrapping both desktop and mobile nav. Hamburger button: `aria-controls="mobile-nav"` + `aria-expanded`. Mobile nav: `id="mobile-nav"`. Focus trap not required (non-modal pattern); ensure Tab order traverses nav links before reaching main content.

## Accessibility Requirements (Phase 2-specific)

All Phase 1 WCAG 2.1 AA requirements inherited. Phase 2 adds:

1. Single `<h1>` per page — hero H1 is the only `<h1>` on the homepage. Section headings use `<h2>`. Step names use `<h3>` within `<h2>` plan section.
2. Trust strip logos: each `<img>` has descriptive `alt` text — `alt="uPerform logo"` etc. Not empty alt (they convey meaning). If rendered as `<svg>`, include `<title>` inside the SVG and `role="img" aria-label="uPerform logo"`.
3. Hamburger button: `aria-label="Open navigation menu"` (closed state) / `aria-label="Close navigation menu"` (open state). Toggle via JS.
4. Nav links: no current page indicator needed for homepage (nav links all go to Phase 3 pages — none is "active" on `/`). When active, add `aria-current="page"`.
5. CTA buttons: each instance of "Book a Fit Call" must have identical, meaningful label. Do not use aria-label to differentiate — context provides disambiguation.
6. Trust strip: `<section aria-label="Background and affiliations">` wrapping the trust strip. Prevents axe-core region landmark error.
7. All homepage sections: wrap in `<section>` with a visually-hidden `<h2>` if the section has no visible heading (e.g., Trust Strip). Use `aria-labelledby` pointing to the visible heading where one exists.
