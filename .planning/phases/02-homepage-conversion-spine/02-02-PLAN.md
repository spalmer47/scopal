**Step A — Create src/components/seo/WebSiteSchema.astro**

This is a new file. Pattern is identical to LegalServiceSchema.astro — read that file first.

```astro
---
import { FIRM } from '../../lib/constants';
const schema = {
  '@context': 'https://schema.org',
  '@type': 'WebSite',
  name: FIRM.legalName,
  url: FIRM.url,
  potentialAction: {
    '@type': 'SearchAction',
    target: {
      '@type': 'EntryPoint',
      urlTemplate: `${FIRM.url}/search?q={search_term_string}`,
    },
    'query-input': 'required name=search_term_string',
  },
};
---
<script type="application/ld+json" set:html={JSON.stringify(schema)} />
```

Why: FIRM.url is used to build the urlTemplate so it reads from constants, not a hardcoded string.
The `{search_term_string}` part is a schema.org template variable — it must remain literally as
`{search_term_string}` in the JSON output (not replaced by JavaScript). Using a template literal
`${FIRM.url}/search?q={search_term_string}` produces the correct string because `{search_term_string}`
is not a JS template variable.
